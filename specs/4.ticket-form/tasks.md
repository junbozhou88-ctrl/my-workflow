## 任务列表

### 功能 1: 表单基础

- [x] T-001: 工单表单组件 + zod schema（邮箱格式/电话正则/联系人必填/描述 5–500），含 label 与 inline 错误 ~30min

### 功能 2: 新增工单

- [x] T-002: 新增弹窗 + 提交联动（成功关闭、refetch、默认待处理） ~30min
- [x] T-003: 30 条上限前端提示与「新增工单」按钮禁用（读 useTickets 数量 + 409 兜底） ~15min

### 功能 3: 编辑工单

- [x] T-004: 编辑弹窗 + 数据回显 + status 下拉字段（待处理/处理中/已完成）+ 提交更新 ~30min

### 功能 4: 失败处理

- [x] T-005: 提交失败处理（弹窗不关闭、保留已填内容、按错误码映射提示、防重复提交） ~15min

## 依赖关系

- T-002 / T-004 依赖 T-001
- T-003 依赖 T-002
- T-005 依赖 T-002、T-004
- 跨 feature：`4.T-002 依赖 2.T-006`（POST）、`4.T-004 依赖 2.T-007`（PUT）、`4.T-003 依赖 2.T-006`（409）；数量与 refetch 来自 `3.T-002`，修改入口来自 `3.T-006` 同级操作列

## 风险点

- 前端 zod schema 必须与 2.ticket-api 保持一致，否则前端通过却被服务端 400
- 编辑回显与 refetch 后的数据一致性：提交成功以服务端返回/refetch 为准
