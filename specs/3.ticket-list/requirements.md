# ticket-list — 需求规格

## 概述

查询并展示当前用户的工单列表，覆盖加载/空/错误/数据四种状态，提供工单删除（含二次确认），并维护「当前用户」浏览器身份。

## 项目信息

- 项目名: my-workflow-lambda
- 架构类型: monorepo，前端 apps/web，对接 2.ticket-api

## 需求版本

| 日期       | 版本 | 说明     |
| ---------- | ---- | -------- |
| 2026-06-17 | v1   | 初始需求 |

## 用户故事

- 作为用户，我想要看到我提交过的工单列表，以便跟进处理进度。
- 作为用户，我想要刷新页面后仍能看到我的工单，且与他人数据隔离。
- 作为用户，我想要删除工单前有确认，以免误删。

## 功能需求

1. [F-001] 生成并持久化 `clientUserId`（localStorage），每次请求带 `x-user-id` 头
2. [F-002] 查询当前用户工单列表（调用 `GET 2.api/tickets`）
3. [F-003] 表格展示字段：联系人、邮箱、电话、描述、状态、创建时间、操作
4. [F-004] 状态以彩色标签展示（待处理/处理中/已完成）
5. [F-005] 四态：加载中、无数据（友好空状态）、请求失败（可重试）、正常数据
6. [F-006] 操作列含「修改」「删除」入口（修改入口联动 feature 4）
7. [F-007] 删除需二次确认，确认后移除并刷新列表，取消则不变
8. [F-008] 删除失败时工单保留并展示明确错误提示
9. [F-009] 移动端表格不严重溢出，操作按钮易点击

## 非功能需求

- 性能: 列表查询有加载反馈，避免请求期间空白
- 安全: 身份标识仅存 localStorage，不含敏感信息
- 兼容性: 桌面与移动端响应式

## 验收标准

- [ ] [AC-001] 首次进入生成 clientUserId 并持久化，刷新后不变
- [ ] [AC-002] 列表正常展示当前用户工单，含全部约定字段
- [ ] [AC-003] 无数据时显示友好空状态而非空白表格
- [ ] [AC-004] 请求中显示加载态，失败显示错误态且可重试
- [ ] [AC-005] 点击删除弹出二次确认；确认后工单移除，取消后不变
- [ ] [AC-006] 删除失败时工单仍在列表并提示错误
- [ ] [AC-007] 移动端表格无严重溢出，操作按钮可点击
